Introduction to Laravel 10

Laravel is a PHP framework designed for building modern web applications with simplicity and elegance. This guide covers the setup and implementation of a basic Laravel CRUD (Create, Read, Update, Delete) application.

Step-by-Step Guide

Step 1: Creating a Laravel 10 Project

  1. Visit the official Laravel Documentation.
  2. Run the following command to create a new Laravel project: 
    composer create-project laravel/laravel example-app

Step 2: Setting Up Your Environment

  1. Ensure Composer is installed. Navigate to the htdocs folder (commonly used with XAMPP).
  2. Open Command Prompt in the htdocs folder and run the command: 
    composer create-project laravel/laravel example-app

Step 2.1: Running Your Project development server

  1. First method -In this we can use two commands for the windows.
    2. Open Command Prompt in the root folder and run the command: 
    php artisan serve
    php -S localhost:8000 -t public
  2. Second method - place your laravel project in the htdocs folder and open our browser and navigate to 
      `http://localhost/your-laravel-app/public`
  3. Or you can do - Or you can clear you cache 
            php artisan config:cache
        php artisan route:cache
        php artisan view:cache
        php artisan cache:clear

Step 3: Configuring the Environment File

  • Rename .env.example to .env in your project directory.
  • Update database credentials: 
    
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=your_database_name
DB_USERNAME=your_username
DB_PASSWORD=your_password

Step 4: Running Database Migrations

Use Laravel's built-in migrations to create database tables:

php artisan migrate

Step 5: Creating a Migration File

Generate a new migration file for the articles table:

php artisan make:migration create_articles_table

Define the table structure in the migration file:


Schema::create('articles', function (Blueprint $table) {
    $table->id();
    $table->string('title');
    $table->text('description');
    $table->string('author');
    $table->timestamps();
});

Step 6: Creating a Controller

Generate an ArticleController to handle CRUD operations:

php artisan make:controller ArticleController

Step 7: Creating Views

  • Create a list.blade.php file in the resources/views directory for displaying articles.
  • Create an add.blade.php file for adding new articles.
  • Create an edit.blade.php file for updating articles.

Step 8: Routing

Define routes in the routes/web.php file for each operation:


use App\Http\Controllers\ArticleController;

Route::get('/articles', [ArticleController::class, 'show']);
Route::get('/articles/create', [ArticleController::class, 'addArticle']);
Route::post('/articles', [ArticleController::class, 'saveArticle']);
Route::get('/articles/{id}/edit', [ArticleController::class, 'editArticle']);
Route::put('/articles/{id}', [ArticleController::class, 'updateArticle']);
Route::delete('/articles/{id}', [ArticleController::class, 'deleteArticle']);

Step 9: Implementing CRUD Methods

  • **Create**: Add an addArticle method in ArticleController for displaying the form.
  • **Read**: Implement a show method to fetch and display all articles.
  • **Update**: Use editArticle and updateArticle methods to handle updates.
  • **Delete**: Implement a deleteArticle method to remove articles.

Step 10: Flash Messages

Redirect users with flash messages after successful operations:


return redirect('/articles')->with('success', 'Article added successfully!');

Enhancements and Best Practices

  • **CSRF Protection**: Always include @csrf in forms to prevent cross-site request forgery attacks.
  • **Validation**: Use Laravel's request validation for form inputs.
  • **Eloquent ORM**: Leverage Eloquent for seamless database operations.
  • **Pagination**: Implement pagination for better readability of large datasets.
  • **Middleware**: Use middleware for authentication and authorization.
  • **Laravel Debugbar**: Install Laravel Debugbar for easier debugging.
